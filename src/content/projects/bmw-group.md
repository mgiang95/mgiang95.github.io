## Problem

Text input in a car prototype is nothing like text input on a phone: reach,
glance time and driving context make every extra keystroke expensive. The
prototype needed an on-screen keyboard that keeps input fast without pulling
attention from the road.

## Approach

As a working student I built a virtual keyboard with dynamic target search —
the layout predicts likely next characters and enlarges their touch targets,
so common inputs need fewer precise taps. Implemented with Apache Cordova,
HTML, CSS, jQuery and JSON so it could run directly inside the prototype
stack.

## Outcome

The keyboard shipped as part of the interior prototype and the target-search
behaviour held up in hands-on evaluations — an early lesson in designing
input systems under constraints rather than screens in isolation.
